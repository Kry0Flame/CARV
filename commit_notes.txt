-----------------------------------------------------------------------
- Extracted all CSS into styles.css; added toast notification styles
- Converted <script> to <script type="module"> — no build step required,
  works as-is on GitHub Pages

- Removed all inline onclick= handlers; all event wiring moved to main.js

DYNAMIC NOZZLE EXTRACTION — removed hardcoded vessel config
-----------------------------------------------------------------------
- Removed hardcoded NOZZLES array (was specific to one vessel drawing)
- Added extractNozzles() in parsers/pdf.js using PDF.js spatial text
  positions to detect nozzle schedule from any drawing:
    1. Groups text items into rows by Y-position proximity
    2. Finds rows containing a nozzle designator (N1, N-1, N1A, etc.)
    3. Pairs each with its orientation angle — prefers values with an
       explicit degree symbol to avoid false matches on pipe sizes /
       pressure ratings; falls back to multiples-of-5 heuristic
    4. Falls back to line-by-line text scan if spatial pass finds nothing
- parsePDF() now returns pdfData.nozzles[]
- validateRun() reads nozzles from pdfData instead of a static import
- End-cut stripping threshold is now dynamic (uses nozzles.length)
- results.js table is dynamic:
    - Nozzles detected  → full comparison table (Nozzle | Print Angle |
      Expected RTP | CNC Angle | Axial X | Status)
    - None detected     → raw CNC cuts table + manual verification prompt
- Banner shows "NO NOZZLE SCHEDULE DETECTED" when extraction finds nothing
- Added .nozzle-sub style for optional nozzle description labels
